### 1. Основание для разработки

Разработка веб-сервиса обусловлена необходимостью автоматизации процессов учёта личных доходов и расходов с целью повышения финансовой грамотности пользователей и предоставления инструментов для анализа и планирования личного бюджета.

---

### 2. Назначение разработки

Создание программного комплекса для автоматизации учёта финансовых транзакций с возможностью категоризации, аналитики, визуализации данных и импорта/экспорта в формате CSV.

---

### 3. Цели разработки

* Создание удобного интерфейса для добавления и управления транзакциями

* Разработка модуля аналитики и визуализации финансовых данных
* Создание инструментов импорта и экспорта данных в формате CSV

#### 4.3. Аналитика и дашборд

* Импорт транзакций из CSV-файла с автоматическим созданием отсутствующих категорий
* Поддержка различных форматов дат при импорте

## 2. Функциональное назначение

Программа предназначена для автоматизации процессов учёта личных финансов и решает следующие задачи:

* Регистрация доходов и расходов с привязкой к категориям и подкатегориям
* Аналитика финансовых данных с фильтрацией по периодам и категориям
* Визуализация динамики доходов и расходов по месяцам
* Импорт транзакций из CSV-файлов сторонних приложений
* Экспорт данных в CSV для дальнейшей обработки

**Функциональные ограничения:** система рассчитана на персональное использование, данные разграничены между пользователями через JWT-аутентификацию.

---

## 3. Описание логической структуры

**Архитектура:** клиент-серверная с разделением на слои.
```
HTTP запрос
    ↓
Middleware (JWT валидация)
    ↓
Handler (обработка запроса, валидация входных данных)
    ↓
Service (бизнес-логика: расчёты, агрегации)
    ↓
Repository (работа с БД через GORM)
    ↓
SQLite (хранение данных)
```

### Backend:
* `cmd/main.go` — точка входа, инициализация роутера и маршрутов
* `internal/handlers/` — HTTP-обработчики (auth, transactions, analytics)
* `internal/service/` — бизнес-логика (расчёт баланса, агрегации, JWT)
* `internal/repository/` — работа с БД, seed категорий
* `internal/models/` — модели данных (User, Category, Transaction, Budget)
* `internal/middleware/` — JWT-аутентификация

### Frontend:
* `static/index.html` — единый файл SPA: авторизация, главная страница, дашборд

### Связи между моделями:
* User (1) → (N) Transaction — у пользователя множество транзакций
* Category (1) → (N) Category — категория может иметь подкатегории (parent_id)
* Category (1) → (N) Transaction — транзакция привязана к категории
* User (1) → (N) Budget — пользователь может устанавливать бюджеты

## 6. API

### Аутентификация
| Метод | Путь | Описание |
|-------|------|----------|
| POST | /api/auth/register | Регистрация |
| POST | /api/auth/login | Вход, получение JWT-токена |

### Транзакции (требуют JWT)
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/transactions | Список транзакций с фильтрами |
| POST | /api/transactions | Создать транзакцию |
| PUT | /api/transactions/:id | Обновить транзакцию |
| DELETE | /api/transactions/:id | Удалить транзакцию |
| GET | /api/transactions/export | Экспорт в CSV |
| POST | /api/transactions/import | Импорт из CSV |

### Категории (требуют JWT)
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/categories | Список категорий и подкатегорий |
| POST | /api/categories | Создать категорию |
| DELETE | /api/categories/:id | Удалить пользовательскую категорию |

### Аналитика (требуют JWT)
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/analytics/summary | Сводка: баланс, доходы, расходы |
| GET | /api/analytics/monthly | Данные по месяцам |
| GET | /api/analytics/budgets | Статус бюджетов |
| POST | /api/budgets | Установить бюджет |
